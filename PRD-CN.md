# 产品需求文档 (PRD)
# Arc Raiders 速查表

**版本：** 1.0
**最后更新：** 2025-11-13
**域名：** arcraiderscheatsheet.org

---

## 1. 执行摘要

### 1.1 产品愿景
打造权威的 Arc Raiders 速查表平台，帮助玩家在战利品管理（保留/出售/回收）、任务进度和工坊制作方面做出最优决策。该平台将成为新手和资深玩家寻求快速、可操作信息的首选资源。

### 1.2 商业机会
- **市场趋势：** "arcraiderscheatsheet" 关键词在 Google 趋势中呈上升趋势，近期搜索量超过 "GPTs"
- **目标市场：** 主要为北美和欧洲
- **变现方式：** Google AdSense 广告
- **竞争优势：** 全面、用户友好的界面，实时数据更新

---

## 2. 市场分析

### 2.1 目标受众

**主要用户：**
- Arc Raiders 玩家（休闲到硬核）
- 寻求物品管理指导的新手玩家
- 需要参考资料的内容创作者
- 贡献游戏知识的社区成员

**用户画像：**
- 年龄：18-35岁
- 地理位置：北美、欧洲（英语市场）
- 游戏经验：中等到高级
- 平台：PC 玩家

### 2.2 竞争格局

**主要竞争对手：**
1. **arctracker.io** - 成熟的追踪器，支持中文
2. **raidercheatsheet.fun** - 社区驱动的速查表

**竞争优势：**
- 基于 Next.js 16 + React 19 构建的简洁现代 UI
- 快速加载时间和响应式设计
- SEO 优化的内容结构
- 移动优先方法
- 专注英语市场

## 3. 产品目标与成功指标

### 3.1 目标
1. 成为 Google SERP 中排名第一的 Arc Raiders 速查表
2. 3个月内实现 10,000+ 月活跃用户
3. 保持 <2秒 页面加载时间
4. 首月内获得 Google AdSense 批准

### 3.2 关键绩效指标 (KPIs)
- **流量：** 月独立访客、页面浏览量、跳出率
- **参与度：** 平均会话时长、每次会话页面数
- **SEO：** 关键词排名、自然流量增长
- **收入：** AdSense CTR、RPM、月收入
- **用户满意度：** 回访率、社交分享

## 4. 核心功能与需求

### 4.1 MVP 功能（第一阶段）

#### 4.1.1 物品数据库与速查表
**优先级：** P0（关键）

**功能：**
- 包含分类的全面物品列表：
  - 武器（按类型、稀有度）
  - 护甲与装备
  - 制作材料
  - 消耗品
  - 任务物品
  - 商贩垃圾

**物品属性：**
- 名称、图标、稀有度等级
- **操作建议：** 保留 / 出售 / 回收
- 价值/价格信息
- 使用说明
- 制作需求（如适用）

**筛选与搜索：**
- 按类别、稀有度、操作建议筛选
- 按物品名称搜索
- 按价值、稀有度、字母顺序排序

#### 4.1.2 任务追踪器
**优先级：** P0（关键）

**功能：**
- 带目标的任务列表
- 每个任务所需物品
- 奖励信息
- 任务链可视化
- 进度追踪（客户端）

#### 4.1.3 工坊与制作指南
**优先级：** P1（高）

**功能：**
- 制作配方数据库
- 材料需求
- 制作站需求
- 输出物品和属性
- 成本效益分析

#### 4.1.4 响应式设计
**优先级：** P0（关键）

**需求：**
- 移动优先设计
- 平板优化
- 桌面体验
- 深色/浅色模式切换
- 快速加载（<2秒初始加载）

### 4.2 第二阶段功能（MVP 后）

#### 4.2.1 用户账户与个性化
- 保存收藏物品
- 个人库存追踪
- 任务进度同步
- 物品自定义备注

#### 4.2.2 社区功能
- 用户提交的技巧和策略
- 物品评分系统
- 评论区
- 社区配装分享

#### 4.2.3 高级工具
- 战利品优化计算器
- 配装规划器
- 资源刷取路线
- 价格趋势追踪

#### 4.2.4 内容更新
- 补丁说明集成
- 新物品提醒
- 元分析文章
- 视频指南集成

## 5. 技术架构

### 5.1 技术栈
- **框架：** Next.js 16（App Router）
- **前端：** React 19、TypeScript
- **样式：** Tailwind CSS v4
- **部署：** Vercel（推荐）或类似平台
- **分析：** Google Analytics 4
- **变现：** Google AdSense

### 5.2 数据管理
**第一阶段（静态）：**
- 物品、任务、配方的 JSON/TypeScript 数据文件
- 静态生成以优化 SEO 和性能

**第二阶段（动态）：**
- 无头 CMS（Sanity/Contentful）用于内容管理
- API 路由用于用户数据
- 数据库（PostgreSQL/MongoDB）用于用户账户

### 5.3 SEO 策略
- 关键页面服务端渲染（SSR）
- 物品/任务页面静态生成
- 语义化 HTML 结构
- 社交分享的 Open Graph 标签
- 富片段的结构化数据（Schema.org）
- XML 站点地图生成
- robots.txt 优化

## 7. 用户体验 (UX) 需求

### 7.1 设计原则
1. **速度优先：** 即时信息访问
2. **清晰度：** 清晰的视觉层次和操作指示器
3. **可访问性：** WCAG 2.1 AA 合规
4. **移动优化：** 触控友好界面
5. **可扫描性：** 易于浏览和查找信息

### 7.2 关键用户流程

**流程 1：快速物品查询**
1. 用户进入首页
2. 使用搜索或浏览分类
3. 找到物品
4. 查看保留/出售/回收建议
5. 根据需要查看更多详情

**流程 2：任务规划**
1. 用户导航到任务板块
2. 按任务类型或进度筛选
3. 选择任务
4. 查看所需物品
5. 与物品数据库交叉参考

**流程 3：制作配方发现**
1. 用户想要制作特定物品
2. 搜索工坊板块
3. 查看配方和材料
4. 检查需要保留哪些材料
5. 规划刷取路线

## 8. 变现策略

### 8.1 Google AdSense 实施
**广告位置：**
- 顶部横幅（728x90 或响应式）
- 侧边栏广告（300x250）
- 内容中广告（板块之间）

**广告优化：**
- 非侵入式放置

- A/B 测试最佳位置
- 懒加载以提升性能

### 8.2 未来收入来源

- 高级功能（无广告、高级工具）

## 11. 风险评估与缓解

### 11.1 风险

**风险 1：游戏更新破坏数据**
- **缓解措施：** 模块化数据结构、版本控制、快速更新流程

**风险 2：来自成熟站点的竞争**
- **缓解措施：** 卓越的 UX、更快的性能、更好的 SEO、社区参与

**风险 3：初期流量低**
- **缓解措施：** 积极的内容营销、社区合作、必要时付费推广

**风险 4：AdSense 拒绝**
- **缓解措施：** 确保优质内容、适当政策、申请前有足够内容

**风险 5：游戏人气下降**
- **缓解措施：** 内容多样化、扩展到相关游戏、建立忠实社区

### 11.2 应急计划
- 通过联盟营销备份变现
- 扩展到其他撤离射击游戏
- 建立邮件列表用于直接沟通
- 创建可下载资源（PDF、应用）

## 14. 待解决问题与决策

1. **数据来源：** 如何维护准确、最新的游戏数据？
   - **建议：** 社区贡献 + 人工验证

2. **CMS vs 静态：** 何时从静态数据过渡到 CMS？
   - **建议：** 从静态开始，在 10k+ 月用户时迁移到 CMS

3. **用户账户：** 是否应在第一阶段实现用户账户？
   - **建议：** 否，先专注核心内容

4. **付费功能：** 是否应提供高级功能？
   - **建议：** 3 个月后根据用户反馈评估

5. **多语言：** 是否应支持其他语言？
   - **建议：** MVP 仅英语，根据流量数据添加语言
